# 消息转发功能需求文档 (PRD) v1.0

## 1. 需求背景与目标
**背景**：现有系统的消息转发功能仅支持展示“待接待状态会话列表”，无法满足运营人员灵活查找特定客户并进行消息分发的需求。
**目标**：优化消息转发流程，支持按**客户昵称模糊搜索**，展示搜索结果及客户当前接待角色（AI、运营、小B、专家），并支持**多选**转发，提升运营效率。

## 2. 功能模块概述
| 模块 | 功能点 | 描述 | 优先级 |
| :--- | :--- | :--- | :--- |
| **入口** | 消息操作菜单 | 在单条消息的操作菜单中增加“转发”按钮 | P0 |
| **弹窗** | 内容预览 | 展示待转发的消息内容（只读） | P0 |
| **搜索** | 模糊搜索 | 支持输入客户昵称关键词进行模糊匹配 | P0 |
| **列表** | 结果展示 | 展示匹配的用户头像、昵称及接待角色标签 | P0 |
| **选择** | 多选机制 | 支持选中多个用户，已选用户以标签形式展示在输入框中 | P0 |
| **操作** | 确认/取消 | 执行转发动作或关闭弹窗 | P0 |

## 3. 详细功能说明

### 3.1 转发弹窗 (Forward Modal)
- **触发条件**：用户点击某条消息的“转发”按钮。
- **UI 布局**：
  - **标题**：“转发消息”
  - **关闭按钮**：右上角“X”图标，点击关闭弹窗。
  - **消息预览区**：
    - 展示原消息文本内容。
    - 样式：背景色区分（如灰色背景），支持长文本滚动查看。
  - **接收人选择区**：
    - 包含一个组合式输入框（Input with Tags）。
    - 包含一个搜索结果下拉列表（Dropdown）。
  - **底部操作栏**：
    - “取消”按钮：点击关闭弹窗。
    - “确认转发”按钮：默认置灰（Disabled），当选中至少 1 位用户后高亮可用。

### 3.2 搜索交互逻辑 (Search Interaction)
- **输入规则**：
  - 用户在输入框中输入关键词（如“张”）。
  - 支持中文、英文、数字输入。
- **触发搜索**：
  - **方式一**：点击输入框右侧的“搜索图标”（放大镜）。
  - **方式二**：在输入框聚焦状态下按“Enter”键。
  - **注意**：仅在上述操作后触发搜索请求，**不支持**输入时实时联想（减少干扰和请求量）。
- **结果展示**：
  - 搜索触发后，在输入框下方展开下拉列表。
  - **无结果**：展示空状态提示（如“未找到匹配用户”图标及文案）。
  - **有结果**：展示匹配的用户列表，列表项高度固定，支持滚动。

### 3.3 用户列表项 (User List Item)
- **展示信息**：
  - **头像**：用户头像缩略图（若无头像显示默认占位符/首字）。
  - **昵称**：用户昵称，高亮匹配的关键词（可选优化点）。
  - **角色标签**：
    - 若用户有当前接待角色，显示对应的彩色标签（无“接待：”前缀字样）。
    - 角色类型包括：`AI`、`运营`、`小B`、`专家`。
    - 若用户无接待角色，**不显示**任何标签，保持清爽。
- **交互状态**：
  - **未选中**：背景白色，鼠标悬停显示浅灰背景。
  - **已选中**：背景高亮（如浅蓝色），右侧显示“对勾”图标。
  - **点击操作**：
    - 点击未选项 -> 选中该用户，输入框中增加对应标签，下拉列表保持展开。
    - 点击已选项 -> 取消选中该用户，输入框中移除对应标签。

### 3.4 选中状态管理 (Selection Management)
- **标签展示**：
  - 已选用户以“标签（Tag/Chip）”形式显示在输入框内部左侧。
  - 标签内容：用户昵称 + “X”删除按钮。
  - 标签样式：背景色需与输入框背景区分。
- **删除操作**：
  - 点击标签上的“X”按钮，移除该用户。
  - 按“Backspace”键（当输入框无文字且光标在最前时），删除最后一个已选标签（可选优化）。
- **清空操作**：
  - 列表底部或右上角提供“清空选择”快捷入口（可选）。

### 3.5 确认转发 (Confirm Action)
- **前置条件**：已选用户列表不为空。
- **执行逻辑**：
  - 点击“确认转发”后，前端收集所有已选用户的 ID (`user_id_list`)。
  - 调用后端接口 `POST /api/message/forward`。
  - 参数：`source_message_id`, `target_user_ids`。
- **反馈**：
  - **成功**：弹窗关闭，顶部提示 Toast “转发成功”。
  - **失败**：保留弹窗，提示 Toast “转发失败，请重试”。

## 4. 异常流程 (Edge Cases)
| 场景 | 表现 | 处理方案 |
| :--- | :--- | :--- |
| **搜索超时/失败** | 接口请求无响应 | 在下拉列表中展示“网络异常，请重试”提示，支持点击重试。 |
| **无匹配用户** | 搜索结果为空 | 下拉列表展示空状态插图及文案。 |
| **输入为空搜索** | 用户未输入直接搜索 | 此时可展示“最近联系人”或提示“请输入关键词”。(原型逻辑为展示所有/推荐列表) |
| **超长消息转发** | 消息内容过多 | 预览区限制最大高度，内部滚动。 |

- **关键视觉点**：
  - 下拉列表紧贴输入框，无多余间隙。
  - 角色标签颜色区分（AI-紫色，运营-蓝色，小B-橙色，专家-绿色）。
  - 弹窗尺寸适中（800px * 600px），保证列表展示空间。
